# Vibe Editor — 동작 규칙 참고 문서

> 이 파일은 앞으로 기능 수정 시 기존 동작을 깨뜨리지 않기 위한 기준점으로 사용한다.  
> 코드 세부사항보다 **어떤 이벤트 → 어떤 동작**의 관점으로 기술한다.

---

## 1. 세션 수명주기

### 세션 시작
- **이미지 업로드**: 파일 선택 시 새 세션이 생성되고, 원본 이미지가 표시된다.
- **텍스트로 생성**: 텍스트 입력 후 전송 시 Gemini 이미지 생성 → 새 세션이 시작된다.
- 세션 시작 후 편집 입력창(ChatPanel)이 나타난다.

### 세션 종료
다음 동작들은 모두 **현재 세션 종료**를 의미하며, 동일한 흐름으로 처리된다:
- **New Session 버튼** 클릭
- **헤더 타이틀(로고)** 클릭
- **닉네임** 변경 (다른 사용자로 전환)

공통 동작:
1. 편집 진행 중(`isLoading`)이면 위 동작이 모두 막힌다.
2. 현재 세션이 있으면 서버에 세션 종료 신호(`/trajectory/end`)를 보낸 뒤 trajectory를 저장한다.
3. 클라이언트 상태(이미지, 히스토리, 채팅 등)를 전부 초기화한다.
4. Sessions 목록을 새로 불러온다.

### 과거 세션 이어보기 (Resume)
- 사이드바에서 과거 세션의 특정 스텝을 클릭하면 **Past View 모드**로 진입한다.
- Past View에서는 해당 스텝의 이미지가 메인 뷰에 표시되고, 편집 입력창은 계속 활성 상태다.
- Past View에서 편집 요청을 보내면: 선택된 스텝 이미지를 기반으로 **새 세션이 생성**되고 편집이 적용된다.
- Resume 후에는 Sessions 목록을 새로 불러온다.

---

## 2. 편집 요청 처리

### 일반 편집 요청
1. 사용자가 채팅 입력창에 텍스트를 입력하고 전송한다.
2. **편집 소스(입력 이미지)**는 사용자가 현재 보고 있는 이미지로 결정된다.
   - 히스토리에서 과거 스텝을 보고 있다면 → 그 스텝의 이미지가 소스
   - 최신 결과를 보고 있다면 → 최신 이미지가 소스
3. **편집 기점(분기점)**은 현재 커서가 위치한 노드다. 최신 결과가 아닌 과거 노드에서 편집하면 그 노드에서 새 브랜치가 생성된다.
4. 서버의 4-에이전트 파이프라인(Planner → Validator → Tool 실행)을 거쳐 결과 이미지가 반환된다.
5. 편집 완료 후: 새 결과로 히스토리가 확장되고, 커서와 활성 브랜치가 새 노드로 이동한다.

### 특수 키워드 처리
- `undo` / `reset` 등 특정 키워드는 4-에이전트 파이프라인을 거치지 않고 즉시 처리된다.
- **undo**: 커서를 이전 노드로 이동한다. 히스토리 트리 구조는 변경되지 않고 커서만 이동한다.
- **reset**: 세션 내 모든 편집을 초기 원본 이미지 상태로 되돌린다.
- undo/reset 후 활성 브랜치(어떤 가지를 보여줄지)는 변경되지 않는다.

### 편집 중 동작
- 편집 요청이 진행 중이면(`isLoading`) 세션 종료/초기화 동작이 모두 막힌다.
- 이미지 뷰어에 로딩 오버레이(spinner)가 표시된다.

---

## 3. 히스토리 탐색

### 커서(Cursor)와 활성 브랜치(Active Branch)
- **커서**: 현재 사용자가 "보고 있는" 노드. 이미지 표시에 사용된다.
- **활성 브랜치**: 사이드바 타임라인에 표시할 경로. 루트 → 현재 브랜치 끝까지의 체인이다.
- 두 개념은 독립적이다: 브랜치 끝까지 이동하지 않고 과거 노드만 볼 수 있다.

### 스크롤 / 스와이프로 히스토리 이동
- **마우스 스크롤** 또는 **터치 스와이프**: 이미지 영역에서만 동작한다.
- 스크롤 위 = 이전(더 오래된) 편집으로 이동
- 스크롤 아래 = 다음(더 최신) 편집으로 이동
- 현재 활성 브랜치의 ancestor chain 내에서만 이동한다 (브랜치 전환 없음).
- Past View 모드에서는 과거 세션의 스텝 목록 내에서 이동한다.
- 히스토리가 1개 이하면 스크롤이 무시된다.

### 사이드바 타임라인 클릭
- 현재 세션의 타임라인 노드를 클릭 → 커서가 해당 노드로 이동하고 해당 이미지가 표시된다. 브랜치는 변경되지 않는다.
- 과거 세션의 스텝을 클릭 → Past View 모드로 진입한다.

### 새 편집 완료 시 자동 동작
- 새 편집 결과가 도착하면 커서와 활성 브랜치가 최신 노드로 자동 이동한다.
- Past View 모드가 활성화되어 있으면 해제된다.
- 히스토리가 2개 이상이 되면 "Scroll / swipe to view previous edits" 힌트가 2.5초 표시된다.

## 4. 브랜치 관리

### 브랜치 생성
- 커서가 과거 노드에 있는 상태에서 새 편집을 하면, 그 노드에서 새 브랜치가 생성된다.
- 기존 브랜치는 삭제되지 않는다. 히스토리 트리에 계속 존재한다.

### +N 배지 (브랜치 분기 표시)
- 타임라인에서 현재 경로 외에 다른 자식이 있는 노드에 `+N` 배지가 표시된다.
- N = 현재 경로에 있지 않은 대안 브랜치의 수.
- 배지 클릭 → 대안 브랜치 목록이 인라인으로 펼쳐진다.

### 브랜치 전환
- 대안 브랜치 항목 클릭 → 해당 브랜치의 가장 마지막 노드로 전환한다.
- 전환 시: 서버에서 최신 트리와 이미지를 가져온 뒤 활성 브랜치, 커서, 표시 이미지를 모두 갱신한다.
- 전환이 완료되기 전까지 캐시된 이미지가 있으면 즉시 표시한다(optimistic).

---

## 5. 이미지 표시 우선순위

다음 순서로 표시할 이미지를 결정한다:

1. **원본 비교 모드** (isHoldingOriginal): 항상 세션의 첫 번째 이미지(업로드/생성 원본)를 표시한다.
2. **Past View 모드**: 선택된 과거 스텝의 이미지를 표시한다.
3. **현재 세션**: 커서가 위치한 히스토리 노드의 이미지를 표시한다. 캐시된 이미지가 없으면 `currentImageB64`(서버에서 가장 최근에 받은 이미지)로 대체한다.
4. 위 모두 해당 없으면 이미지 없음 상태(빈 화면)를 표시한다.

### 원본 비교
- **마우스**: 이미지 영역에서 누르는 동안(mousedown) 원본을 보여주고, 떼면(mouseup/mouseleave) 되돌아온다.
- **터치**: 이미지 영역에서 손가락을 대는 순간 원본을 보여준다. 30px 이상 스와이프가 감지되면 원본 비교가 취소되고 히스토리 탐색 모드로 전환된다.

---

## 6. 사이드바 (Sessions / History)

### 현재 세션
- 현재 편집 중인 세션은 사이드바 하단 "Current Session" 섹션에 표시된다.
- 활성 브랜치의 루트 → 현재 끝 노드까지 세로 타임라인으로 표시된다.
- 현재 커서가 있는 노드가 강조 표시된다.

### 과거 세션 목록
- 현재 세션을 제외한 과거 세션들이 위에 나열된다.
- 세션 헤더 클릭 → 해당 세션의 편집 히스토리가 펼쳐진다.
- 과거 세션도 브랜치 구조를 표시하며, 활성 브랜치 전환을 지원한다.

### 레이블 표시
- 각 히스토리 노드의 레이블은 편집 의도(intent)의 앞 5단어로 축약하여 표시한다.
- 전체 텍스트는 hover 시 tooltip으로 확인할 수 있다.

### 표시 조건
- 사이드바는 화면 너비 md(768px) 이상에서만 표시된다.

---

## 7. 닉네임 / 사용자 관리

- 앱 진입 시 닉네임을 입력해야 한다. 닉네임은 Sessions 목록 필터링에 사용된다.
- 헤더의 닉네임 클릭 → 닉네임 인라인 편집 가능.
- 닉네임 변경 시: 현재 세션이 종료되고 상태가 전부 초기화된다. 새 닉네임의 Sessions 목록을 불러온다.

---

## 8. 페이지 스크롤 제어

- **페이지 전체 스크롤은 항상 비활성화**한다.
- 스크롤 이벤트는 오직 이미지 영역에서의 히스토리 탐색에만 사용된다.
- 이미지 영역 외부의 스크롤 이벤트는 모두 무시한다.
- 스크롤/터치 이벤트 핸들러는 앱 최초 마운트 시점에 단 한 번 window에 등록한다.

---

## 9. 입력창 동작

### 세션 없음 (초기 화면)
- 이미지 업로드 버튼과 텍스트 입력창이 제공된다.
- 텍스트 전송 시: Past View가 활성화된 상태면 해당 스텝에서 Resume → 새 세션 생성. 아니면 텍스트→이미지 생성으로 새 세션 시작.

### 세션 있음 (ChatPanel)
- 텍스트 전송 시 일반 편집 요청이 처리된다.
- 편집 중에는 입력창이 비활성화된다.
